Presentation

AeroNav est une application web de gestion d'un service de navettes entre un aeroport et des hotels. Elle permet d'enregistrer les reservations de clients a leur arrivee, de gerer la flotte de vehicules et les hotels, puis d'optimiser automatiquement l'affectation des voitures aux clients en regroupant les trajets compatibles.

Le coeur de l'application est un moteur de planification qui, pour une date donnee, repartit les reservations dans les vehicules disponibles selon des regles de capacite, de disponibilite et de proximite.

Fonctionnalites

  • Reservations : liste, creation et suppression des reservations clients.
  • Hotels : CRUD complet (nom, adresse, distance par rapport a l'aeroport).
  • Voitures : CRUD de la flotte (marque, nombre de places, type de carburant).
  • Distances entre hotels : gestion des distances utilisees pour le regroupement.
  • Planning : visualisation des groupes de reservations et des voitures attribuees pour une date.
  • Optimisation : allocation automatique des vehicules selon l'algorithme metier.

Algorithme d'allocation

L'allocation des vehicules repose sur trois parametres constants :

  • Temps d'attente : 30 minutes (delai avant qu'une voiture reparte / soit liberee).
  • Vitesse moyenne : 20 km/h.
  • Distance de proximite : 5 km (seuil pour regrouper deux hotels).

Principe general

  1. Recuperer et trier les reservations par heure d'arrivee (ordre croissant).
  2. Pour chaque reservation, chercher les voitures disponibles et liberees a temps.
  3. Filtrer par capacite : au moins le nombre de personnes requis en places libres.
  4. Selectionner la meilleure voiture selon les criteres ci-dessous.
  5. Mettre a jour l'etat du vehicule : places restantes et heure de liberation (heure d'arrivee + 30 min).

Criteres de selection (dans l'ordre)

  1. Voiture avec la plus petite capacite suffisante.
  2. En cas d'egalite, celle qui a deja roule (plus de trajets).
  3. En cas d'egalite, preference de carburant (Diesel prioritaire).
  4. En dernier recours, choix aleatoire.

Regroupement de clients

Un client peut etre ajoute a une voiture deja affectee si toutes ces conditions sont reunies :

  • il reste assez de places ;
  • son heure d'arrivee est anterieure a heure de liberation + temps d'attente ;
  • il vise le meme hotel, ou un hotel situe a moins de 5 km.

Stack technique

  • Backend : Java 8+, Spring Boot 2.7.18, Spring Framework 5.3.33, Spring MVC.
  • Persistance : JPA (javax 2.2), Hibernate 5.6.15, MySQL 8.
  • Vues : Thymeleaf, HTML5, CSS3, JavaScript (Fetch API).
  • Build : Maven.

Demarrage

Creer la base de donnees MySQL puis lancer l'application :

CREATE DATABASE reservationnavettejpa;

mvn clean install
mvn spring-boot:run

La configuration se trouve dans src/main/resources/application.properties (port 8082, identifiants MySQL). L'application est alors accessible sur http://localhost:8082/.
